# 作者拟合引擎 · 产品文档

## 这是什么

一个内容资产工程化系统。把一个人的文章、直播稿、知识星球帖拆成可复用材料，提取他的判断方式，然后批量生成判断力接近他的新文章和短视频文案。

**不是仿写。不是学口癖。是提取判断方式。**

---

## 你能得到什么

```
输入：一个人的内容资产（≥50篇）
  ↓
引擎运转
  ↓
输出：
  1. 材料池（可检索、可调用的五维材料库）
  2. 判断规则集（可迁移到新选题的生成规则）
  3. 校准后的生成模板（你的风格 + 决策树路由）
  4. 样稿（3-5篇验证稿 + 读者数据）
```

---

## 怎么用

### 第一步：准备

- 一个人的内容资产，≥50 篇（公众号文章+直播稿+星球帖都行）
- 确认这个人的署名范围（同一账号下可能有多个作者——只取目标作者）

### 第二步：让 Agent 跑

打开 Claude Code 或 Codex，加载 `author-fit-engine` skill。Agent 会自动：

1. 审计内容量
2. 清洗来源（署名归一+污染门禁）
3. 拆解为五维材料（问题/概念/观点/案例/方案）
4. 提取三层拟合（他怎么选材料、怎么连接、怎么取舍）
5. 跑三维检验（判断有没有力、结构有没有推进、表达有没有毛刺）
6. 生成 3-5 篇验证稿
7. 跑发布管线（对标→内容审计→钩子优化→AI检测→标题优化→配图）

### 第三步：你来判断

Agent 不会替你做判断。你只在两个地方需要介入：
- 选题时——选方向。这是你的行业认知。
- 管线验收时——选钩子和标题。这是你的文字审美。

其余全部自动。

### 第四步：发布 + 看数据

文章发到公众号。数据回来（阅读、转发、私信），Agent 校准决策树。

## 你需要什么

- 有内容积累（≥50篇自己的文章）
- 有基本网感（能判断「像不像」）
- 有一个 Agent 运行环境（Claude Code / Codex / Cursor）

---

## 你不能用它做什么

- 零内容积累——引擎空转
- 完全自动——你需要判断选题和验收
- 保证爆款——引擎是概率系统，不是魔法
