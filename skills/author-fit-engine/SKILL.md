# author-fit-engine：作者拟合引擎

你不是在仿写。不是在学口癖。是把一个人的内容拆到最小单元，再反过来验证——用这些单元能不能生成判断方式接近他的新内容。

**你交付的不是几篇文章。是一套可复用的提取引擎。**

---

## 一句话定义

给定一个人的内容资产，提取他的判断方式，批量生成接近他判断力的文章和短视频文案。换一个人，引擎不变。

---

## 核心结构

### 四层架构

```
内容资产层：我有什么材料？    → 五维拆解 + 来源清洗
作者表达层：他怎么判断？      → 三层拟合 + 三维检验 + 决策树
生产编排层：怎么生成稿子？    → 门禁 + 生成 + High-Fi 开关
验收反馈层：这东西能不能用？  → 管线验收 + 人在回路 + Phase 6
```

### 五维材料

```
QST  问题单元  — 文章要处理什么问题？
CON  概念单元  — 用什么工具解释？
OPI  观点单元  — 核心判断是什么？
CAS  案例单元  — 用什么证据支撑？
SOL  方案单元  — 怎么收束、怎么让人行动？
```

铁律：五维是内容资产单位，不是文章结构模板。

### 三层拟合

```
材料拟合 — 原文/池子里用了哪些材料？
关系拟合 — 材料之间怎么连接？因果？递进？反转？
选择拟合 — 舍弃了什么？为什么？

三层拟合是五维→作者表达的桥。不得跳过。不得压缩成"风格拟合"。
```

### 三维检验

```
深层判断 — 为什么这样看问题？不是摘要，不是中庸。
中层结构 — 如何推进读者？入口→转折→反驳→收束。
表层风格 — 有没有毛刺？段落长短是否交替？

三维是贯穿装配全过程的检验面，不是最后喷漆。
```

## 来源清洗

```
署名归一 → 五桶分桶 → 污染门禁
只有目标作者 confirmed 的文章能进入作者表达拟合。
无署名、转载、课程销售类 → author_fit_allowed=false
```

---

## 工作流程

### Phase 1：审计

判断内容量是否够：≥50 篇目标作者文章。不够→不进入重工程。

### Phase 2：拆解

从原文提取 QST/CON/OPI/CAS/SOL。建立材料池。

### Phase 3：理解

对目标作者的 5-10 篇核心文章跑三层拟合 + 三维检验。提取决策树——什么题材选择什么手法。

### Phase 4：验证

生成 3-5 篇样稿。发布。收集读者反馈（阅读、转发、私信）。反馈回来校准决策树。

### Phase 5：量产

决策树校准后，选题→生成→管线验收→发布。

## 生成合约

每篇稿子在生成前必须回答以下问题。缺一个，不输出正文。

### 生成前

```
[ ] 五维材料确认：QST/CON/OPI/CAS 来源？
[ ] 三层拟合：材料/关系/选择 各一行？
[ ] 三维检验：深层判断/中层结构/表层风格 各一行？
[ ] 决策树路由：题材是什么？开头/意象/收束策略？
```

### 生成后

```
[ ] 字数 ≥ 800
[ ] 可能吧格式：:::intro + :::highlight
[ ] 配图 prompt 在同一文件末尾
[ ] 原文验收卡：至少 1 篇原文作为 gold sample 对照
[ ] Phase 6（一行）：本轮 agent_mistake？next_rule？
```

## 禁止项

以下行为在任何情况下不得执行：

```
- 未经用户验收直接发布
- 跳过三层拟合或三维检验
- 把五维材料写成文章段落模板
- 概念改名（QST→问题段落、CON→解释段等）
- 突破四层架构范围新增系统层
- 把低反馈包装成"再发几篇试试"
- 用 README 美化替代工程验收
```

## 下一 Agent 门禁

下游 Agent（排版 Agent、发布 Agent、管线 Agent）在开工前必须确认：

```
[ ] Phase 6 已填写？
[ ] 用户已说"过"？
[ ] 管线验收(benchmark→content→hook→AI→title)全部通过？
[ ] 原文验收卡已完成？
[ ] 生成合约 checklist 全部打勾？

缺任何一个，下游 Agent 不得进入执行。
```

## 发布管线

```
选题（用户定方向）
  ↓
生成稿（Agent 走合约门禁）
  ↓
管线验收：
  benchmark + content → 静默
  hook → 用户选
  AI-check → 有命中才报
  title → 用户选
  ↓
用户说"过" → 发布稿（正文 + 配图 prompt 同一文件）
  ↓
用户贴可能吧 → 发
```

---

## 决策树（通用模板）

不同作者需要重新提取决策树。以下是树林项目验证通过的决策点结构：

```
决策点1：开头策略（对抗→硬砸 / 自省→场景代入 / 揭露→重复三次）
决策点2：CAS密度（防御高→子弹式 / 防御低→内心独白）
决策点3：意象使用（抽象判断→需要意象 / 具体判断→可跳过）
决策点4：收束策略（逼认→极硬 / 接纳→接纳式 / 揭露→反思式）
决策点5：元反思触发（判断堆积≥3层→需要 / 自省型→不需要）
```

## 人在回路

你的判断在两个节点：
1. 选题时 → 行业认知和网感
2. 管线 hook/title 时 → 文字审美

其余由 Agent 在合约约束下自动完成。

---

## 停止规则

- 同一篇修订 ≥2 轮未通过 → 归档失败
- 用户反馈「不像」「没击中」→ 停止自动修订
- 单篇经验 ≠ 全局规则

---

## 当前限制

本 skill 不覆盖以下场景。后续 Agent 不得在这些缺口自行扩展：

```
- 短视频视频文件生成（当前仅到文案）
- 多作者混合拟合
- 非中文内容
- 自动多平台分发
- 配音/配乐
```

缺口只标记。不偷扩。
