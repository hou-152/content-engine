# Content Engine — 作者拟合引擎

> 给定一个人的内容资产，提取他的判断方式，批量生成接近他判断力的文章和短视频文案。
> 换一个人，引擎不变。树林是第一个样本。

## 架构

```
四层：
  内容资产层 → 作者表达层 → 生产编排层 → 验收反馈层

五维：
  QST / CON / OPI / CAS / SOL

三层拟合：
  材料拟合 → 关系拟合 → 选择拟合

三维检验：
  深层判断 / 中层结构 / 表层风格

六系统 Loop：
  概念冻结 / 工程契约 / 内容资产 / 作者表达 / 生产编排 / 验收反馈
```

## 快速开始

### 安装

```bash
# Claude Code
npx skills add <repo-url> --skill author-fit-engine

# Codex
/plugin marketplace add <repo-url>
/plugin install author-fit-engine
```

### 跑第一个样本

```
/author-fit-engine

Agent: 你有 ≥50 篇目标作者的文章吗？署名范围确认了吗？
你：有。XX的公众号文章，署名是XX。
Agent: 审计通过。开始拆解...
```

Agent 会自动完成：来源清洗 → 五维拆解 → 三层拟合 → 三维检验 → 生成 3-5 篇验证稿。

## 人在回路

你的判断在两个节点：
1. **选题时** → 行业认知和网感
2. **管线 hook/title 时** → 文字审美

其余由 Agent 在合约约束下自动完成。
