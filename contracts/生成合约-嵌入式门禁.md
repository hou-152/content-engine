# 生成合约：嵌入式门禁

日期：2026-06-17
用途：每次生成稿子时嵌入 prompt。缺一个不输出正文。

---

## 生成前必答（三组门禁）

### 第一组：五维材料确认

```
[ ] QST 从哪来？（池子里的哪个问题单元，或新提取）
[ ] CON 从哪来？（池子里的哪个概念单元）
[ ] OPI 从哪来？（核心判断——来源或原创）
[ ] CAS 从哪来？（场景/案例——来源或原创）
[ ] SOL 方向是什么？

缺任何一个，不进入正文。
```

### 第二组：三层拟合

```
[ ] 材料拟合（一行）：原文/池子里用了哪些材料？
[ ] 关系拟合（一行）：材料之间怎么连接？因果？递进？反转？
[ ] 选择拟合（一行）：舍弃了什么？为什么？

缺任何一个，不进入正文。
```

### 第三组：三维检验

```
[ ] 深层判断：为什么这样看问题？不是摘要，不是中庸。
[ ] 中层结构：如何推进读者？入口→转折→反驳→收束。
[ ] 表层风格：有没有毛刺？段落长短是否交替？有没有「太顺」的地方？

缺任何一个，不进入正文。
```

---

## 生成后必答（发布门禁）

```
[ ] 决策树路由：题材是什么？开头/意象/收束策略是否正确？
[ ] 字数 ≥ 800？
[ ] 可能吧格式：:::intro + :::highlight？
[ ] 配图 prompt：在同一文件末尾？
[ ] Phase 6（一行）：本轮 agent_mistake？next_rule？
```

---

## 使用方式

把这个文件的内容嵌入到生成 prompt 里。不是「先读了再写」——是写在 prompt 里，当成不能跳过的步骤。

```
你是树林内容资产系统的撰稿 Agent。

在生成任何正文之前，你必须先回答以下问题。缺一个，不输出正文。

[合约内容]
```
