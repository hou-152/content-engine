# 发布管线：人在回路合约

日期：2026-06-17
来源：执行问题1-3的修法落地

---

## 问题 → 修法

```
问题1：发的时候忘记贴配图 → 发布稿末尾直接带配图prompt
问题2：人不清楚在哪判断 → 管线分步停。两步静默，三步停
问题3：管线没当验收   → 你说了"过"，才生成发布稿
```

---

## 管线执行规范

### 静默步（自动跑，不占用你的注意力）

```
benchmark → 已知对标树林，自动通过。只报异常。
content  → 自动跑。只报异常（AI味/结构问题）。
```

### 停等步（给你选项，你选）

```
hook     → 给3个开头选项。你选一个。不替你选。
AI-check → 0命中 → 一行带过。≥1命中 → 展开位置和严重度，问"修吗"。
title    → 给3-5个标题选项（公众号+小红书）。你选。
```

### 验收门禁

```
管线跑完 → 问"过吗"
  过   → 生成发布稿（正文 + 配图prompt在同一文件内）
  不过 → 停在不过的那一步，只修那一步
```

---

## 发布包模板

每篇发布稿固定包含：

```
正文（可能吧格式：:::intro + :::highlight）
排版入口：https://knb.im/mp/
配图 Prompt（公众号封面 + 小红书封面，中英文各一组）
```

缺任何一项，不发。

---

## 人在回路的位置

```
选题（你定方向）
  ↓
生成稿
  ↓
管线：benchmark(静默) → content(静默) → hook(你选) → AI-check(异常报) → title(你选)
  ↓
你："过"
  ↓
生成发布稿（正文 + 配图prompt 同一文件）
  ↓
你最后看一遍 → 贴可能吧 → 发
```

你的判断在两个点：
1. 选题时——你的行业认知和网感
2. 管线 hook/title 时——你的文字审美

其余由 Agent 在合约约束下自动完成。
